\chapter{Conclusiones y recomendaciones}

\section{Conclusiones}

El prototipo obtenido utilizando el procesamiento de imágenes por medio de redes neuronales convolucionales demuestra ser efectivo identificando fallos en los \textit{balloons} (globos) de catéteres cardiacos con un alto índice de efectividad. El cual es satisfactorio con respecto a los requerimientos.

Una vez obtenido el resultado que arroja el programa se procede manualmente a promover la unidad a la siguiente fase en la linea de producción o a desecharla en el lugar de forma satisfactoria.

El prototipo permite realizar la manipulación (puesta en posición y toma de fotografías) de forma no destructiva (sin causar daños estructurales). Pues este cumple con las regulaciones interiores referentes a la manipulación de unidades de \textit{balloon}.

El sistema integrado final permite realizar todo el proceso de reconocimiento y detección de fallos satisfactoriamente y le indica al usuario si la unidad es defectuosa.

Actualmente el proceso manual realizado es destructivo, y este conlleva realizar la ruptura de una muestra representativa del lote de unidades. Esto significa que se pierden unidades comerciales y se presume que los demás miembros del lote tendrán el mismo comportamiento. Esto aunque sea efectivo estadísticamente, impide que sea posible obtener una representación general de la totalidad del lote de unidades, lo cual impide que se puedan realizar mejoras minuciosas al proceso de forma efectiva.

Al automatizar un proceso de forma precisa, ágil y confiable se consigue no sólo incrementar la producción y reducir costes. También se eliminan incertidumbres propias del ser humano como: cambios de animo, negligencia, cansancio, distracciones, necesidades fisiológicas, etc.

Un método de inspección no destructivo permite reducir el desperdicio de material y unidades, incrementando la ganancia y la capacidad de producción.



\section{Recomendaciones}

A futuro es importante realizar pruebas de campo más exhaustivas, para garantizar sesiones de trabajo más extensas y exigentes para limar asperezas antes de una incorporación al proceso de producción.

Para un desarrollo futuro es importante realizar la producción de unidades completamente nuevas y pasarlas por el prototipo. Esto para tener una población de prueba aún más extensa y aumentar las probabilidades de obtención de todos los casos posibles para las unidades.

Este proyecto abre una puerta para incluir diferentes procesos de inspección visual. Provee la infraestructura necesaria para posteriormente desarrollar algoritmos con otros objetivos y estos ser incluidos en la misma herramienta. Por eso es importante no abandonar la constante mejora de este proyecto.
